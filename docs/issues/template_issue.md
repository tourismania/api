## Scope

### Домен (`internal/domain/`)

- `entity/[name].go` — `[Entity]{ Field1, Field2, ... }`. *Явно указать какие поля required (не указатели), какие optional (указатели) — и почему.*
- `enum/[name]_status.go` — *если применимо.*
- `repository/[name]_repository.go` — интерфейс: `Store`, `FindByX`, `List(Filter)`, `Update`, `SoftDelete`/`Delete`. `[Name]Filter{ ... }`.
- `service/[name]_manager.go` — *единый менеджер жизненного цикла, инварианты, проверка владения. Явно указать: есть ли роль-based bypass (обычно НЕТ), какой сентинел возвращается при несовпадении владельца (`NotFound` vs `Forbidden` — выбор влияет на то, раскрывается ли существование чужой записи).*
  - `service.Actor{ ... }` — *identity, передаваемая в домен из presentation-слоя. Указать какие поля required int, какие *int, и обосновать через инвариант из «Контекста».*

### Infrastructure (`internal/infrastructure/persistence/postgres/`)

- `repository/[name]_repository.go` — реализация `domain/repository`.
- `queries/[name].sql` (+ `make sqlc`) — *все чтения фильтруют `deleted_at IS NULL`, если есть soft delete.*
- `mapper/[name]_mapper.go` — маппинг `model.[Name] ↔ entity.[Name]`.

### Application (CQRS, `internal/application/`)

**command:**

- `create_[name]`, `update_[name]`, `delete_[name]`. *Identity write-side передаётся явно (`CurrentUserID int`, `[Owner]ID int` — required, заполняется presentation-слоем строго из `CurrentUser`, другого источника нет).*

**query:**

- `get_[name]s` — список. `Query{ ... }`.
- `get_[name]` — приватное получение одного. `Query{ ID, [Owner]ID int }` — *required, если ручка только для авторизованных.*
