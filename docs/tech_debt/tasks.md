# Технический долг

- [ ] Rate-limiter в `router.go` намеренно применяется ко всему `/api/v1`, а не только к `/airports`. Это временное упрощение — при ревью не считать это находкой.
- [ ] `UpdateOffer` (`OfferManager.Update` → `OfferRepository.Update` → `db.UpdateOffer`) — потенциальный lost update при конкурентных `PATCH`. `UpdateOfferParams.Title`/`Description`/`Status` — не указатели: SQL безусловно перезаписывает все три колонки значением, которое `OfferManager.Update` смержил в память (существующее значение + непустые поля запроса) на момент чтения offer, а не только реально изменённые поля. Два параллельных `PATCH` с непересекающимися полями (A меняет `title`, B меняет `status`) могут привести к тому, что изменение одного из них потеряется — тот, кто закоммитил вторым, перезапишет колонку первого своим стале-снапшотом. Решение — оптимистическая блокировка (`WHERE uuid = $1 AND updated_at = $expected`, конфликт → отдельный сентинел/`409`) или переход на частичный `UPDATE` с `COALESCE`/nullable-параметрами на уровне SQL. Не исправлено в рамках issue №12 — обнаружено при code review, воспроизводимость требует двух конкурентных запросов, отсутствует тест на гонку. Issue: [№19](https://github.com/tourismania/api/issues/19).
